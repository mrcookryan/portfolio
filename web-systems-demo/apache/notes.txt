# Apache Notes — Web Systems Demo

These notes provide additional context for how Apache behaves in real environments and how it supports the rest of the Web Systems Demo.


# Virtual Hosts
Apache uses Virtual Hosts to serve multiple sites from the same server.
Each vhost defines:
- DocumentRoot
- ServerName / ServerAlias
- Logging
- Directory permissions
- Routing behavior

See apache.conf for examples.


# .htaccess Behavior
.htaccess files allow per-directory overrides.
Common uses:
- URL rewriting
- Access control
- Security headers
- Redirects

Note: AllowOverride must be enabled for .htaccess to function.


# Rewrite Engine
mod_rewrite enables flexible URL routing.
Key directives:
- RewriteEngine On
- RewriteCond (conditions)
- RewriteRule (pattern → destination)

See rewrite-rules.conf for practical examples.


# Security Headers
Security headers help mitigate common web vulnerabilities:
- X-Frame-Options
- X-Content-Type-Options
- Referrer-Policy
- Permissions-Policy
- Content-Security-Policy (CSP)
- Strict-Transport-Security (HSTS)

See security-headers for a complete header set.


# HTTPS / SSL
Apache typically uses:
- SSLEngine On
- SSLCertificateFile
- SSLCertificateKeyFile

HSTS should only be enabled once HTTPS is fully configured.


# Relationship to DNS / Hosting / Routing
Apache sits directly after DNS resolution:

- DNS → points the domain to the server  
- Hosting → provides the environment Apache runs in  
- Apache → serves the correct site via Virtual Hosts  
- Routing → handled via mod_rewrite or application-level routing  

Apache is the core of the Web Systems Demo’s server-side behavior.
