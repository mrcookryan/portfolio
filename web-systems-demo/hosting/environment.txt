# Hosting Environment — Web Systems Demo

This file provides a representative example of a typical hosting environment used for small business sites, VPS deployments, and Apache-based web systems.


# Operating System
Ubuntu Server 22.04 LTS
- Long-term support
- Stable package versions
- Common choice for Apache/Nginx hosting


# Web Server
Apache HTTP Server 2.4.x
Modules commonly enabled:
- mod_rewrite
- mod_headers
- mod_ssl
- mod_expires
- mod_deflate
- mod_proxy (optional)
- mod_proxy_http (optional)

Alternative server:
Nginx 1.22+ (not used in this demo but referenced in hosting-overview.md)


# PHP / Runtime
PHP 8.1+
Enabled extensions:
- php-json
- php-mbstring
- php-xml
- php-curl
- php-gd
- php-mysql (if using a database)

Other runtimes (optional):
- Node.js 18+
- Python 3.10+


# File System Layout
/var/www/example.com/
/var/www/example.com/public_html/
/var/www/example.com/logs/
/var/www/example.com/backups/
/etc/apache2/sites-available/
/etc/apache2/sites-enabled/

Permissions:
- Files: 644
- Directories: 755
Owner:
- www-data (Apache)
- root (system-level)


# SSL / Certificates
Let’s Encrypt (Certbot)
Files:
- fullchain.pem
- privkey.pem
- chain.pem

Renewal:
- Automatic via systemd timer


# Logging
Apache logs:
- /var/log/apache2/access.log
- /var/log/apache2/error.log

System logs:
- /var/log/syslog
- /var/log/auth.log


# Monitoring
Basic uptime monitoring:
- Ping-based checks
- HTTP status checks
- Optional: UptimeRobot / HetrixTools


# Deployment Method
Typical workflow:
- Git pull or SFTP upload
- Validate Apache config (apachectl configtest)
- Reload Apache (systemctl reload apache2)
- Check logs for errors
- Run deployment checklist


# Relationship to Web Systems Demo
This environment supports:
- DNS resolution → server IP
- Apache virtual hosts → domain routing
- Rewrite rules → clean URLs and API paths
- Hosting structure → file organization, permissions, logs

It represents the underlying system that ties all other components together.
